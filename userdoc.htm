User Documentation

These artifacts highlight my profiencies at task oriented instruction and information mapping, as well as my ability to use multiple types of documentation authoring software such as MadCap Flare, Framemaker, InDesign, and Microsoft Office.

Manuals

CTE Registration System Manual [pdf] The Instructional Design Unit and the Center for Teaching Excellence worked in the same office at SPSU. Because my boss has background in information technology, when the CTE needed a new registration system she took on the project. Outside consultants developed the system using a list of requirements written by another student assistant. I wrote the manual entirely on my own. This 57 page user manual covers every possible task for every role in the system. I designed this document using InDesign. Spring 2013

Mini Clock Cam [pdf] The Mini Clock Cam is a portable alarm clock with a hidden camera inside. I rewrote the manual in Microsoft Word to address poor readability and document design. I designed the layout so that the manual could fit in the small rectangular box. Mini Gadgets, Spring 2012

DoorSpy Manual [png] The DoorSpy is an entry detection device that notifies users when the door it is to attached is opened. I rewrote the manual in Microsoft Word to address poor readability and document design. I designed the layout so that the manual could fit in the small rectangular box. Mini Gadgets, Spring 2012

Information Mapping Project [pdf]In this document, I have taken content from the Georgia Department of Drivers’ Services Drivers Manual and organized it into Information Maps. I redid most of the graphics, but some of the text is the same, as the purpose of this exercise was to learn the formatting prescribed by Information Mapping. User Documentation, Fall 2012
